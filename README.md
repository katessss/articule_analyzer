# AI-ассистент для мета-анализа

**AI-powered система для автоматизированного поиска, анализа и извлечения данных из научных статей.**

Комплексное решение для работы с PDF-документами: извлечение мультимодальных данных (текст, таблицы, изображения) с применением OCR-технологий и последующий интеллектуальный анализ через языковые модели (LLM) для структурированного извлечения ключевых признаков из научного контента.

## Возможности

### 🔍 Интеллектуальный поиск статей
- Итеративный поиск уникальных научных публикаций через Perplexity API
- Автоматическая фильтрация дубликатов
- Сохранение результатов в CSV с названиями и ссылками

### 📄 Комплексная обработка PDF-документов
- Извлечение текста, таблиц и метаданных из PDF
- Поддержка OCR-технологий для сканированных документов
- Обработка сложных многостраничных документов
- Распознавание различных форматов научных статей

### 🤖 AI-powered извлечение данных
- Использование Google Gemini и OpenAI для интеллектуального анализа текста
- Настраиваемая схема извлечения ключевых признаков
- Пакетная обработка признаков для оптимизации затрат
- Автоматическое retry при ошибках парсинга

### 📊 Структурированный вывод
- Экспорт в CSV с поддержкой Unicode
- Итеративное добавление данных в существующие файлы
- "Широкий" формат таблицы для удобного статистического анализа

### 🖥️ Гибкие режимы работы
1. **Streamlit Web-интерфейс** — интуитивный графический интерфейс для быстрой работы
2. **Консольное приложение** — для автоматизации и интеграции в скрипты
3. **Программный API** — для полного контроля и кастомизации процесса

## Установка

### Требования
- Python 3.8+
- API-ключи:
  - Perplexity AI (для поиска статей)
  - Google AI / Gemini (для анализа текста)
  - OpenAI (опционально, как альтернатива Gemini)

### Шаг 1: Клонирование репозитория
```bash
git clone <repository-url>
cd articule_analyzer-main
```

### Шаг 2: Установка зависимостей
```bash
pip install -r requirements.txt
```

### Шаг 3: Настройка переменных окружения

Создайте файл `.env` в корневой директории:

```env
# Perplexity API для поиска статей
PPLX_API_KEY=your_perplexity_api_key_here

# Google AI / Gemini для анализа текста
GOOGLE_API_KEY=your_google_api_key_here

# OpenAI (опционально)
OPENAI_API_KEY=your_openai_api_key_here
```

**Получение API-ключей:**
- Perplexity: https://www.perplexity.ai/settings/api
- Google AI: https://makersuite.google.com/app/apikey
- OpenAI: https://platform.openai.com/api-keys

## Использование

### Веб-интерфейс (Streamlit)

```bash
streamlit run app.py
```

**Workflow:**
1. **Шаг 1**: Введите тему исследования → Система найдет статьи
2. **Шаг 2**: Укажите путь к папке с PDF → Настройте признаки для извлечения
3. **Шаг 3**: Получите результаты в формате CSV

**Особенности:**
- Динамическое добавление/удаление признаков через UI
- Возможность пропустить Шаг 1 и загрузить свои PDF
- Логи в реальном времени
- Прогресс-бары для длительных операций

### Консольный режим

```bash
python console_app.py
```

Интерактивный режим с пошаговыми инструкциями:
1. Ввод темы для поиска
2. Указание пути к папке с PDF
3. Автоматический анализ и сохранение результатов

### Пррограммный режим

Для полной автоматизации отредактируйте `core_logic.py`:

```python
from core_logic import run_links, run_analyzing

# 1. Поиск статей
run_links(
    TOPIC="Влияние биочара на почву",
    total_iterations=3
)

# 2. Анализ PDF
extraction_schema = {
    "Название статьи": "Название статьи",
    "Год выпуска": "Год публикации (только число)",
    "Тип почв": "Тип почв на исследуемом участке",
    # ... добавьте нужные признаки
}

run_analyzing(
    pdf_folder_path="./статьи",
    features=extraction_schema,
    batch_size=4  # Количество признаков в одном запросе к AI
)
```

## Ограничения и рекомендации

### Ограничения
- **PDF качество**: Сканированные документы плохого качества могут распознаваться с ошибками
- **Структура статей**: Нестандартное форматирование может снизить точность извлечения
- **Языковые модели**: Качество зависит от обучающих данных модели
- **API лимиты**: Учитывайте rate limits и стоимость API-запросов

### Рекомендации
1. **Batch size**: Начните с 2-4 признаков на батч для баланса скорость/точность
2. **Промпты**: Чем конкретнее формулировка признака, тем точнее результат
3. **Валидация**: Всегда проверяйте выборку результатов вручную
4. **Токены**: Используйте Google Gemini Flash для экономии (~20x дешевле GPT-4)
5. **Логи**: Следите за логами для отладки проблем с извлечением
