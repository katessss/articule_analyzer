# Папка для временных файлов

Эта папка используется для хранения промежуточных данных при обработке PDF.

## Содержимое

### Извлеченный текст из PDF
- `<имя_файла>_extracted_content.txt` - текстовое содержимое каждого PDF

**Формат:**
```
--- Page 1 ---

Текст первой страницы...

| Таблица | в | markdown | формате |

--- Page 2 ---

Текст второй страницы...
```

## Назначение

1. **Кэширование:** Текст извлекается из PDF один раз и сохраняется
2. **Повторное использование:** При смене схемы извлечения текст не извлекается заново
3. **Отладка:** Можно проверить качество извлечения текста вручную

## Оптимизация

### Экономия времени и API-запросов
При повторном запуске с новой схемой:
```python
# Первый запуск - извлекает текст
run_analyzing("./pdfs", schema1, batch_size=4)

# Второй запуск - использует кэш из temp/
run_analyzing("./pdfs", schema2, batch_size=4)  # Быстрее!
```

### Проверка извлеченного текста
```bash
# Просмотр извлеченного текста
cat temp/article_name_extracted_content.txt

# Поиск по содержимому
grep "keyword" temp/*.txt
```

## Очистка

Периодически очищайте папку для экономии места:

```bash
# Удалить все временные файлы
rm -rf temp/*

# Linux/Mac
find temp/ -name "*.txt" -mtime +30 -delete  # Старше 30 дней

# Windows PowerShell
Get-ChildItem temp -Filter *.txt | Where-Object {$_.LastWriteTime -lt (Get-Date).AddDays(-30)} | Remove-Item
```

## Размер файлов

Типичные размеры:
- Обычная статья (10-20 страниц): 50-200 KB
- Большая статья (50+ страниц): 500 KB - 2 MB
- Папка с 100 статьями: ~10-50 MB

**Рекомендация:** Очищайте temp/ после завершения проекта.
